### 운영체제가 하드웨어를 관리

프로그램이 직접 하드웨어를 관리하긴 위험해서 **운영체제가 중간관리자 역할**을 한다.

**하드웨어가 자원을 안정하고 효율적으로 나눠쓰게 해주는게 OS 그러니까 운영체제이다.**

OS 부분을 읽으면서 **추상화**라는 단어를 많이 읽게 됐는데, 이 추상화를 통해 안전하고 단순히 컴퓨터를 다룰 수 있게 도와주는 거 같다.

운영체제의 핵심 임무는:
1.  어떤 프로그램이 메모리(다른 프로그램)를 건드리지 못하게 **보호** 하고,
2.  CPU 시간과 메모리 디스크에 **자원을 효율적으로** 나눠주며,
3.  하드웨어를 쉽게 다룰 수 있게 **인터페이스**를 제공해준다.



### 시스템은 네트워크로 서로 통신한다

오늘 날 프로그램 대부분은 **다른 컴퓨터와 연결**되어 있다.
예) 인터넷 브라우저, 메신저, 게임 등 네크워크가 있어야 동작한다.

네트워크는 컴퓨터 사이의 데이터를 주고 받는 통로로 데이터를 주고 받을 때도 **비트와 나열**을 전송한다.

통신구조의 쉬운 예로 컴퓨터와 구글 서버의 통신으로 예를 들어보면:
> 브라우저에 `google.com` 입력 -> 브라우저가 네트워크를 통해 **요청(request)**를 보냄
> -> 구글 서버가 **응답(response)**을 보내줌 -> 화면에 구글이 뜸

정도가 되겠다.



### 중요 주제들

*   **암달의 법칙**
    *   성능 개선은 전체 중 병목이 되는 부분을 얼마나 개선하느냐에 달려있다.

*   **동시성과 병렬성**
    *   **동시성 (Concurrency)** : 여러 작업을 겉보기에는 동시에 진행하는 것 (사실은 번갈아가면서 진행)
    *   **병렬성 (Parallelism)** : 여러 CPU 코어가 **정말 동시에** 다른 작업을 진행하는 것

*   **추상화**
    *   복잡한 컴퓨터 시스템을 단순한 개념으로 감싸서 사용하게 만든 것
